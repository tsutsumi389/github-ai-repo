# Architecture Decision Records (ADR)

本ディレクトリでは、プロジェクトのアーキテクチャに関する決定を記録・管理する。

## 目的

- 設計判断の経緯と理由を永続的に記録する
- AIエージェントが設計意図を正しく理解するための権威的情報源とする
- 過去の決定がなぜ行われたかを追跡可能にする

## ステータス定義

| ステータス | 意味 |
|-----------|------|
| **Accepted** | 承認済み。現在有効な決定 |
| **Superseded** | 置換済み。新しいADRに取って代わられた |
| **Deprecated** | 廃止。もはや適用されない |

## 不変原則

一度書いたADRの内容は**書き換えない**。決定を変更する場合は：

1. 新しい番号のADRを作成する
2. 新ADRの `supersedes` フィールドに旧ADRのIDを記載する
3. 旧ADRの `status` を `Superseded` に、`superseded_by` に新ADRのIDを記載する

ステータスの変更のみが、既存ADRに対して許可される唯一の編集である。

## 新規ADR作成手順

1. `template.md` をコピーする
2. ファイル名を `ADR-NNN-<短い説明>.md` とする（NNNは連番、ゼロ埋め3桁）
3. フロントマターを記入する
4. 各セクションを埋める
5. `bun run check:adr` でバリデーションを実行する

## バリデーション

```bash
bun run check:adr
```

フロントマターの必須フィールドやステータス整合性を検証する。
