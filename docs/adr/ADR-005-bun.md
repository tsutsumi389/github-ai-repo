## コンテキスト

パッケージマネージャーおよびランタイムとして、npm、pnpm、Yarn、Bun が候補となった。

## 決定

Bun を採用する。

## 理由

新しいプロジェクトの機会に Bun を試用する目的で採用した。加えて、以下の技術的利点がある。

- **TypeScript の直接実行**: トランスパイルなしで `.ts` ファイルを実行できるため、スクリプト（ADRバリデーション等）の実行が手軽
- **パッケージインストールの高速さ**: npm と比較してインストール速度が大幅に速い

## 結果

- Node.js との互換性が完全ではない場合があり、一部パッケージで問題が生じる可能性がある
- 本番実績が npm / pnpm と比べて少ないため、問題発生時の情報が限られる
- Node.js 依存が強い機能で互換性問題が発生した場合は再検討が必要
